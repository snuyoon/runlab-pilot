## 주요 의사결정과 근거

1. **하이브리드(WKWebView 셸 + 네이티브 알람)** — 설문/관리자/백엔드를 웹으로 유지해 앱 심사 없이 즉시 수정 배포. 네이티브는 웹이 못 하는 알람만. App Store 4.2(웹뷰 래퍼 리젝)는 AlarmKit이라는 실질 네이티브 기능으로 방어.
2. **AlarmKit (iOS 26.1+)** — 유일하게 "앱 꺼져도 + 무음 관통 + 잠금화면 알람 UI"를 공식 보장. 26.1 기준인 이유: 26.0의 `stopButton` 이니셜라이저가 26.1에서 deprecated → 현행 API로 작성. 미만 기기는 반복 로컬알림 폴백(제한적).
3. **OSTRC는 검증 도구 그대로** — 핵심 4문항 문구/선택지/점수 불변(논문 권고). 드래그 요청은 SnapSlider(4선택지 스냅)로 수용, 1~10 변환은 사용자에게 트레이드오프 설명 후 기각됨. 게이트키퍼·반복연결·시간손실 근거는 [OSTRC.md](OSTRC.md).
4. **기상 설문 1~10 드래그** — 자체 문항이라 척도 변경 자유(1~5→1~10, 2026-07-02, 서버 데이터 초기화와 함께 적용).
5. **Neon Postgres** (vs Supabase) — 사용자 선택. 기존 Vercel 계정 마켓플레이스 연동으로 신규 계정 불필요, 무료 티어로 파일럿 충분.
6. **참여 코드 사전등록 + 무작위 접미사** — 열거·이웃 오타로 인한 데이터 오염 방지 (리뷰에서 확정된 취약점).
7. **RPE·기상설문 하루 1회** — 사용자 지시. RPE는 "그날의 대표 세션" 개념.
8. **스누즈 미구현** — AlarmKit 스누즈는 위젯 익스텐션 필수라 범위 제외. 필요해지면 ActivityKit 위젯 추가.
9. **가민(Forerunner 265 Music) 세션 연동 — Apple 건강(HealthKit) 경유로 구현(2026-07-03)**. 리서치·검증(멀티에이전트 워크플로) 후 결정: FR265 → Garmin Connect(Apple 건강 공유 ON) → HealthKit → iOS 네이티브 셸이 러닝 워크아웃(거리·페이스·시간·심박) 읽어 브리지로 웹 → `/api/sync` → Neon. **무료·가민 파트너십 불필요·기존 셸 재사용**이라 파일럿 최적. 구현: `HealthKitService.swift`(권한·HKObserverQuery 백그라운드 전달·30일 catch-up), `HealthKitBridge.tsx`(루트 레이아웃 상주 수신), `WorkoutSession`(kind `workout`, UUID 멱등), dashboard/admin 표시·CSV. **HealthKit capability 추가 = 다음 빌드용**(현 심사 빌드엔 없음). 실기기 엔드투엔드(러닝→Connect→건강→앱) 최종 확인 필요. iOS 전용(안드로이드 별도).

## 적대적 리뷰에서 배운 교훈 (재발 방지)

세 차례 멀티에이전트 리뷰로 확정 버그 22건 수정. 다시 하기 쉬운 실수들:

- **"UI가 예약됐다고 표시" ≠ "실제 예약됨"** — 네이티브 동기화는 로그인·홈 진입에서 반드시 초기 실행. 알람 상태를 바꾸는 새 경로를 만들면 `nativeSyncAlarms` 호출 여부 확인.
- **AlarmKit `.custom` 보조 버튼은 알람을 멈추지 않는다** — 인텐트에서 `stop(id:)` 직접 호출.
- **비직렬 Task로 시스템 자원(알람) 조작 금지** — AlarmService의 직렬화 체인 유지.
- **마이그레이션 판정은 병합 후 값이 아니라 parsed 원본으로** — 기본값 병합이 끼어들면 분기가 죽는다.
- **`ON CONFLICT DO NOTHING`은 갱신 패턴과 충돌** — 부분 전송→갱신이 필요하면 DO UPDATE.
- **웹 알람의 분(===) 정확일치 판정 금지** — 타이머 정지 후 재개되면 영영 안 울림 → 목표 timestamp 비교.
- **Wake Lock은 hidden 되는 순간 해제** — visibilitychange에서 재획득.
- **`AnimatePresence mode="wait"` 금지** (백그라운드 탭 rAF 동결로 다음 화면 미마운트).
- **하루 1회 설문은 페이지 자체에서 가드** — 홈 버튼만 막으면 딥링크/뒤로가기로 뚫림.
- 리뷰 워크플로 스크립트는 세션 디렉토리에 저장돼 재사용 가능 (Workflow scriptPath 재호출).

   - ⚠️ **스크린샷 1장뿐**(로그인 화면, 6.7"/APP_IPHONE_67로 업로드). 로그인 후 화면은 시뮬레이터 웹뷰에 코드 입력이 필요한데 **macOS 손쉬운 사용 권한 미부여로 합성 클릭/키입력이 차단**돼 자동 캡처 불가. 권한 부여 시(시스템 설정>개인정보>손쉬운 사용) 재캡처 가능. 정식 심사는 로그인 화면만으로 리젝 위험 → 승인 전 보강 권장.
