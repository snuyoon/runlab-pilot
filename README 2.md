# RunLab — AI 스마트 러닝워치 연구 참여자 컴플라이언스 관리 앱

> 러닝 연구 참여자의 수면 워치 착용률을 극대화하고 기상 EMA(Ecological Momentary Assessment) 데이터를 수집하기 위한 게이미피케이션 기반 컴플라이언스 관리 앱의 **컨셉 데모**입니다.

VC(벤처캐피탈) 및 연구 교수님 대상 피칭을 위해 제작된 프로토타입으로, 실제 모바일 브라우저에서 동작합니다.

---

## 라이브 데모

**https://runlab-demo.vercel.app**

스마트폰 브라우저에서 접속하거나, PC에서 개발자 도구 > 모바일 뷰로 확인하세요.  
연구 참여 코드는 아무 값이나 2자 이상 입력하면 됩니다 (예: `SNU-001`, `TEST`).

## 주요 기능

### 1. 로그인 — 연구 참여 코드 입력
연구 안내문을 통해 발급받은 고유 코드(예: `SNU-001`)를 입력합니다. 최초 진입 시 러닝 파트너 선택 화면으로, 이후에는 홈으로 바로 이동합니다.

### 2. 러닝 파트너 선택
강아지, 고양이, 토끼, 여우, 곰 중 하나를 선택합니다. 선택한 동물은 참여자와 함께 성장하는 게임 캐릭터가 됩니다.

### 3. 메인 홈 — 캐릭터 + 미션 + 현황
- **동물 캐릭터**: 레벨에 따라 알 → 새끼 → 청소년 → 성체로 진화, 애니메이션으로 생동감 표현
- **XP 게이지**: 오늘의 미션 완료 시 XP 적립, 레벨업 달성
- **연속 참여 일수**: 연속으로 참여한 날짜 수 표시
- **오늘의 미션**: 기상 EMA 설문 / 워치 착용 확인 / 러닝 기록 동기화
- **이번 주 착용 현황**: 요일별 워치 착용 성공/실패 시각화
- **취침 시작 버튼**: 수면 플로우 진입
- **시연 패널**: 레벨업/퇴화/XP 추가/초기화 버튼 (VC 시연용)

### 4. 알람 설정
기상 알람 시간과 취침 리마인더 시간을 휠 방식으로 설정합니다. 예상 수면 시간이 자동 계산됩니다.

### 5. 수면 플로우 — 4단계 자동 전환
```
[취침 준비] → [수면 중] → [알람 울림] → [알람 해제 + EMA 자동 이동]
```
- 취침 시작 버튼 → 수면 모드(어두운 화면, 시계 희미하게 표시)
- 시연용 "알람 울리기" 버튼으로 즉시 알람 페이즈로 전환
- 밀어서 알람 끄기(드래그 슬라이더) → EMA 화면으로 자동 이동

### 6. 기상 EMA 설문
알람 해제 직후 자동 실행되는 3문항 이모지 설문입니다.
- 어젯밤 수면의 질 (😫 ~ 😄)
- 근육통/피로감 (😵 ~ 🤸)
- 오늘의 기분 (😞 ~ 🤩)

제출 완료 시 캐릭터 축하 애니메이션과 함께 +30 XP가 적립됩니다.

### 7. 관리자 대시보드
연구자용 모니터링 화면입니다. (더미 데이터 기반)
- 전체 참여자 수 / 평균 착용률 / 위험 참여자 수 요약 카드
- 주간 착용률 추이 바 차트
- 참여자별 착용률 + EMA 응답률 + 연속 참여일 테이블
- 위험(30% 미만) / 경고(50% 미만) 참여자 필터링

### 파일/디렉토리 역할 상세

| 파일 | 역할 |
|---|---|
| `src/app/layout.tsx` | Geist 폰트 로드, 모바일 최적화 뷰포트 설정, 앱 메타데이터 선언 |
| `src/app/globals.css` | `.mobile-frame` 클래스 (360×780 고정 모바일 프레임), Tailwind CSS 설정 |
| `src/app/page.tsx` | 연구 참여 코드 입력 + 유효성 검사 + 동물 선택 여부에 따라 라우팅 |
| `src/app/select/page.tsx` | 5종 동물 카드 선택 UI, 선택 확정 시 `gameStore`에 저장 |
| `src/app/home/page.tsx` | 캐릭터 카드(XP바, 레벨, 스테이지), 미션 목록, 주간 착용현황, 시연 패널 포함 |
| `src/app/alarm/page.tsx` | `TimeWheel` 컴포넌트로 시/분 선택, 예상 수면 시간 계산, `gameStore`에 저장 |
| `src/app/sleep/page.tsx` | `Phase` 타입 4단계(`bedtime` → `sleeping` → `alarm` → `dismiss`)로 화면 전환 |
| `src/app/ema/page.tsx` | 3문항 이모지 5점 척도, 전체 응답 시 제출 활성화, 제출 후 XP 적립 + 홈 이동 |
| `src/app/dashboard/page.tsx` | 더미 참여자 10명 데이터 기반 테이블 + 착용률 바 차트 + 상태별 필터 |
| `src/store/gameStore.ts` | `GameState` 인터페이스, `loadState`/`saveState`/`resetState` 함수, 동물 정의(`ANIMALS`), 스테이지 계산(`getStage`), XP 임계값(`xpForLevel`) |

## 로컬 실행 방법

### 요구 사항
- Node.js 18 이상
- npm / yarn / pnpm / bun

### 설치 및 실행

```bash
# 저장소 클론
git clone <repo-url>
cd runlab-demo

# 의존성 설치
npm install

# 개발 서버 실행
npm run dev
```

브라우저에서 [http://localhost:3000](http://localhost:3000) 접속 후, 개발자 도구(F12)에서 모바일 기기(iPhone 14 Pro 권장)로 전환하면 실제 앱과 동일한 화면을 확인할 수 있습니다.

```bash
# 프로덕션 빌드
npm run build
npm run start
```

## 사용자 플로우

### 참여자 플로우 (일반 사용자)

```
앱 최초 진입
    │
    ▼
[/] 연구 참여 코드 입력
    │
    ├─ 동물 미선택 ──▶ [/select] 러닝 파트너 선택 ──▶ [/home]
    │
    └─ 동물 선택완료 ──────────────────────────────▶ [/home]

[/home] 메인 홈
    │
    ├─ "취침 시작하기" ──▶ [/sleep] 수면 플로우
    │       │
    │       ├─ 취침 준비 화면 (워치 착용 안내)
    │       ├─ 수면 중 화면 (어두운 화면, 희미한 시계)
    │       ├─ 알람 화면 (진동 애니메이션, 밀어서 끄기)
    │       └─ 해제 완료 ──▶ [/ema] EMA 설문 자동 진입
    │                               │
    │                               └─ 3문항 응답 + 제출 ──▶ +30 XP + [/home]
    │
    ├─ "설정 변경" ──▶ [/alarm] 알람/취침 시간 설정
    │
    └─ "관리자 보기" ──▶ [/dashboard] 관리자 대시보드
```

### 캐릭터 성장 시스템

```
레벨  1~2  : 알 (🥚)
레벨  3~4  : 새끼 (예: 🐶)
레벨  5~7  : 청소년 (예: 🐕)
레벨  8~10 : 성체 (예: 🦮)

XP 획득 조건
  - 기상 EMA 제출    : +30 XP
  - 시연 패널 XP 추가: +30 XP (시연용)

레벨업 임계값: 레벨 N × 50 XP (예: Lv.3 진입에 100 XP 필요)

패널티: 7일 연속 미참여 시 레벨 1(알)로 퇴화
```
